775. 全局倒置与局部倒置

难度中等

给你一个长度为 n 的整数数组 nums ，表示由范围 [0, n - 1] 内所有整数组成的一个排列。

全局倒置 的数目等于满足下述条件不同下标对 (i, j) 的数目：

  • 0 <= i < j < n
  • nums[i] > nums[j]

局部倒置 的数目等于满足下述条件的下标 i 的数目：

  • 0 <= i < n - 1
  • nums[i] > nums[i + 1]

当数组 nums 中 全局倒置 的数量等于 局部倒置 的数量时，返回 true ；否则，返回 false 。

示例 1：

输入：nums = [1,0,2]
输出：true
解释：有 1 个全局倒置，和 1 个局部倒置。

示例 2：

输入：nums = [1,2,0]
输出：false
解释：有 2 个全局倒置，和 1 个局部倒置。

提示：

  • n == nums.length
  • 1 <= n <= 5000
  • 0 <= nums[i] < n
  • nums 中的所有整数 互不相同
  • nums 是范围 [0, n - 1] 内所有数字组成的一个排列

解法一：树状数组求逆序对数量

剑指 Offer 51. 数组中的逆序对

​ 在数组中的两个数字，如果前面一个数字大于后面的数字，则这两个数字组成一个逆序对。输入一个数组，求出这个数组中的逆序对的总数。

class Solution {
    public boolean isIdealPermutation(int[] nums) {
        int n = nums.length;
        int count1 = 0;
        int[] count = new int[n+1];
        long count2 = 0;
        update(count,nums[0]+1);
        for(int i = 1; i < n; i++){
            if(nums[i] < nums[i-1]){
                count1++;
            }
            count2 += i - get(count,nums[i]);
            update(count,nums[i]+1);
        }
        return count1 == count2;
    }

    void update(int arr[], int k){
        while(k < arr.length){
            arr[k]++;
            k += k&(-k);
        }
    }
    int get(int arr[],int k){
        int ans = 0;
        while(k > 0){
            ans += arr[k];
            k -= k &(-k);
        }
        return ans;
    }
}

解法二：暴力+归并排序

[Java] 三种解法简洁题解 暴力 & 前缀和 & 数学归纳

  • 解法一：暴力法
  • 直接暴力求解题目的全局倒置和局部倒置。求全局倒置即逆序对的个数，可以使用归并排序求解。
  • 时间复杂度：\(O(nlogn)\)
  • 空间复杂度：\(O(n)\)
class Solution {
    int n, p, g;
    public boolean isIdealPermutation(int[] nums) {
        n = nums.length;
        for (int i = 0; i < n - 1; i++) if (nums[i] > nums[i + 1]) p++;
        merge(nums, 0, n - 1); 
        return g == p;
    }
    void merge(int[] nums, int l, int r) {
        if (l >= r) return;
        int mid = (l + r) >> 1;
        merge(nums, l, mid);
        merge(nums, mid + 1, r);
        int[] tem = new int[r - l + 1];
        int i = l, j = mid + 1, k = 0;
        while (i <= mid && j <= r) {
            if (nums[i] <= nums[j]) tem[k++] = nums[i++];
            else {
                g += mid - i + 1;
                tem[k++] = nums[j++];
            }
        }
        while (i <= mid) tem[k++] = nums[i++];
        while (j <= r) tem[k++] = nums[j++];
        for (i = l, k = 0; i <= r; i++, k++) nums[i] = tem[k];
    }
}

解法三：前缀和

  • 根据题目全局倒置和局部倒置的定义发现，要两者数量相等，那么对于当前数i,nums[i] > nums[i+1],那么此时nums[i]必须小于等于i+1后面所有数，否则就会全局倒置的个数就会多于局部倒置。因此，对于每个位置i来说，若当前位置的数nums[i] 大于后缀最小值，那么直接返回false，若所有位置都满足，那么代表两者数量相等，返回true。
  • 时间复杂度：\(O(n)\)
  • 空间复杂度：\(O(1)\)
class Solution {
    public boolean isIdealPermutation(int[] nums) {
        int n = nums.length, minv = nums[n - 1];
        for (int i = n -  3; i >= 0; i--) {
            if (nums[i] > minv) return false;
            minv = Math.min(minv, nums[i + 1]);
        }
        return true;
    }
     
}

解法四：数学归纳

  • 解法三：数学归纳 0 <= nums[i] < n,并且nums 中的所有整数 互不相同，那么数组中的数必然是0到n-1随机排列。

    通过解法二我们知道，要返回true代表对于i位置的全局倒置来说除了局部倒置外不能存在其他的逆序对。

    若, nums[i] > nums[j], j > i + 1,那么代表存在其他的逆序对。

    对于0来说，若它的下标大于1,那么必然会有额外的逆序对产生。

    对于1来说，若它的下标大于2，那么也会有额外的逆序对产生。 ...

    对于x来说，若它的下标大于x+1,那么也会有额外的逆序对产生。

    通过以上可以发现nums中的数所对应的下标不能超过其特定的值，否则必然会有额外的逆序对产生。

    同理也可以归纳出，若某个数的坐标小于x-1,那么也会产生额外的逆序对。

    因此，对于数nums[i]，其位于的坐标范围位[nums[i] - 1, nums[i] + 1]

    即 nums[i] - 1 <= i <= nums[i] +1, 即 abs(i - nums[i]) <= 1，若abs(i - nums[i]) > 1则不合法

class Solution {
    public boolean isIdealPermutation(int[] nums) {
        for (int i = 0; i < nums.length; i++) if (Math.abs(i - nums[i]) > 1) return false;
        return true;
    } 
}

1712. 将数组分成三个子数组的方案数

难度中等89

我们称一个分割整数数组的方案是 好的 ，当它满足：

  • 数组被分成三个 非空 连续子数组，从左至右分别命名为 left ， mid ， right 。
  • left 中元素和小于等于 mid 中元素和，mid 中元素和小于等于 right 中元素和。

给你一个 非负 整数数组 nums ，请你返回 好的 分割 nums 方案数目。由于答案可能会很大，请你将结果对 109 + 7 取余后返回。

示例 1：

输入：nums = [1,1,1]
输出：1
解释：唯一一种好的分割方案是将 nums 分成 [1] [1] [1] 。

示例 2：

输入：nums = [1,2,2,2,5,0]
输出：3
解释：nums 总共有 3 种好的分割方案：
[1] [2] [2,2,5,0]
[1] [2,2] [2,5,0]
[1,2] [2,2] [5,0]

示例 3：

输入：nums = [3,2,1]
输出：0
解释：没有好的分割方案。

提示：

  • 3 <= nums.length <= 105
  • 0 <= nums[i] <= 104

题解来自：https://leetcode.cn/problems/ways-to-split-array-into-three-subarrays/solution/by-user2269a-gwgc/#%E6%96%B9%E6%B3%95%E4%B8%80%EF%BC%9A%E4%BA%8C%E5%88%86%E6%90%9C%E7%B4%A2

一种简单的思路是枚举第一分割点 a，然后对每个第一分割点枚举第二分割点 b 并验证是否满足条件 left <= mid <= right, 当然这样的时间复杂度是 O(n^2)，对于本题而言会超时。所以必须思考更快的方法。 再进一步思考条件，首先因为左边的不等式 left <= mid，mid 不能太小，有明确的下界；其次因为右边的不等式 mid <= right = tot - left - mid 变形可得 mid <= (tot - left) / 2, 所以 mid 不能太大（太大就会超过 left )，有明确的上界。同时由于所有元素都是非负的，所以 mid 关于 b 是不严格单调递增的，就可以想到对于某个第一分割点 a = i，若存在合法的若干 b ，必然都位于一个连续的区间 [j, k] 上，且这个区间上的每一个 b 的取值都对应一种同时满足上下界条件的切割方法。累加该区间的长度到结果中就可以了。

res += k - j + 1;

方法一：二分搜索

对任意 a = i，分别各用一次二分搜索来求出 j, k, 总的时间复杂度是 \(O(nlogn)\)

class Solution {

    public static final int MOD = (int) (1e9 + 7);

    public int waysToSplit(int[] nums) {
        int n = nums.length;
        // 前缀和
        int[] p = new int[n + 1];
        for (int i = 0, t = 0; i < n; i++) {
            t += nums[i];
            p[i + 1] = t;
        }
        long res = 0;
        int tot = p[n];
        for (int i = 1; i < n - 1 && p[i] * 3 <= tot; i++) {
            int left = p[i];
            // 二分搜索求下限
            int l = i + 1, r = n - 1;
            while (l <= r) {
                int mid = l + r >> 1;
                if (p[mid] - left < left) l = mid + 1;
                else r = mid - 1;
            }
            int j = l;
            // 二分搜索求上限
            l = i + 1;
            r = n - 1;
            while (l <= r) {
                int mid = l + r >> 1;
                if (p[mid] - left <= tot - p[mid]) l = mid + 1;
                else r = mid - 1;
            }
            int k = r;
            // 累加到结果中
            res += k - j + 1;
        }
        return (int) (res % MOD);
    }
}

方法二：双指针

观察示例 2，在这个示例中，随着 i 增大，j, k 也各自增大。这就提醒我们用双指针的方法来优化时间复杂度。

i = 1, j = 2, k = 3;
i = 2, j = 3, k = 4;

假设 j 对 i 单调递增且k 对 i 单调递增, 那么 i, j和 i, k 分别视作一对双指针。在 i 得递增过程中，每次都从上一轮循环中得到 j 和 k 的位置继续向后搜索，而不进行重置，这样虽然一轮循环中 j++ 和 k++ 可能会执行多次，但是整个过程中 j, k 只会对每个元素访问一次，所以总的时间复杂度是 \(O(N)\).

class Solution {

    public static final int MOD = (int) (1e9 + 7);

    public int waysToSplit(int[] nums) {
        int n = nums.length;
        // 前缀和
        int[] p = new int[n + 1];
        for (int i = 0, t = 0; i < n; i++) {
            t += nums[i];
            p[i + 1] = t;
        }
        long res = 0;
        // 双指针
        for (int i = 1, j = 2, k = 2; i < n - 1 && p[i] * 3 <= p[n]; i++) {
            int left = p[i];
            // 两个分割点必须保持前后顺序
            j = Math.max(i + 1, j);
            while (j < n - 1 && p[j] - p[i] < left) j++;
            while (k < n - 1 && p[k + 1] - p[i] <= p[n] - p[k + 1]) k++;
            // line 21 : 不能使用 if (k < j) break; 提前结束循环
            res += k - j + 1;
        }
        return (int) (res % MOD);
    }
}

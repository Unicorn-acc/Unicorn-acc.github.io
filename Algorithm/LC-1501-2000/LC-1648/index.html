LC-1648.销量价值减少的颜色球

1648. 销售价值减少的颜色球

难度中等76

你有一些球的库存 inventory ，里面包含着不同颜色的球。一个顾客想要 任意颜色 总数为 orders 的球。

这位顾客有一种特殊的方式衡量球的价值：每个球的价值是目前剩下的 同色球 的数目。比方说还剩下 6 个黄球，那么顾客买第一个黄球的时候该黄球的价值为 6 。这笔交易以后，只剩下 5 个黄球了，所以下一个黄球的价值为 5 （也就是球的价值随着顾客购买同色球是递减的）

给你整数数组 inventory ，其中 inventory[i] 表示第 i 种颜色球一开始的数目。同时给你整数 orders ，表示顾客总共想买的球数目。你可以按照 任意顺序 卖球。

请你返回卖了 orders 个球以后 最大 总价值之和。由于答案可能会很大，请你返回答案对 109 + 7 取余数 的结果。

示例 1：

输入：inventory = [2,5], orders = 4
输出：14
解释：卖 1 个第一种颜色的球（价值为 2 )，卖 3 个第二种颜色的球（价值为 5 + 4 + 3）。
最大总和为 2 + 5 + 4 + 3 = 14 。

示例 2：

输入：inventory = [3,5], orders = 6
输出：19
解释：卖 2 个第一种颜色的球（价值为 3 + 2），卖 4 个第二种颜色的球（价值为 5 + 4 + 3 + 2）。
最大总和为 3 + 2 + 5 + 4 + 3 + 2 = 19 。

示例 3：

输入：inventory = [2,8,4,10,6], orders = 20
输出：110

示例 4：

输入：inventory = [1000000000], orders = 1000000000
输出：21
解释：卖 1000000000 次第一种颜色的球，总价值为 500000000500000000 。 500000000500000000 对 109 + 7 取余为 21 。

提示：

  • 1 <= inventory.length <= 105
  • 1 <= inventory[i] <= 109
  • 1 <= orders <= min(sum(inventory[i]), 109)

二分法

思路1：优先队列

要销售额最大，显然每次卖最贵（剩余数量最多）的那个球。 容易想到把球放进大顶堆，每次卖堆顶的那个球。 时间复杂度 O(orders * logN)，由于 orders 太大，这种解法超时。

思路2：二分查找

假如卖完orders个球，剩余的最大同色球数量是 remainMax，那么超过 remainMax 的球肯定都要卖掉，即卖掉（初始数量 － 剩余数量）个该种球。如果此时 orders 不为零还能卖，就在剩余数量是 remainMax 的几种球里，随便挑哪几种球卖掉，直到 orders 为零。

流程

二分搜索［剩余的最大同色球数］

a. 超过［剩余的最大同色球数］的都卖掉
b. 等于［剩余的最大同色球数］的，随便挑几个卖掉，直到orders为零。

返回收益 a + b
class Solution {
    int MOD = (int)1e9 + 7;
    public int maxProfit(int[] inventory, int orders) {
        if (inventory == null || inventory.length == 0) return 0;

        int max = 0;
        for (int in : inventory)
            max = Math.max(max, in);
        
        // 用二分查找找出卖完额度之后，剩余最大（同色）球数量是多少
        // 用 lo, hi 来逼近 “剩余最大（同色）球数量”
        int lo = 0, hi = max; 
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (getBallCnt(inventory, mid, orders) > orders) // mid 取小了，导致卖多了
                lo = mid + 1;
            else hi = mid;
        }
        // 跳出循环就找到卖掉 orders 个球，剩余的最大（同色球）数量
        int remainMax = lo;

        long res = 0L;  // 卖球得到的收益

        // a: 初始数量 > 剩余数量的都卖掉
        for (int cnt : inventory) {
            if (cnt <= remainMax) continue;
            // 卖掉该球的价值依次是[cnt, cnt - 1, cnt - 2,..., remainMax + 1], 剩下的 remainMax 就不卖了
            // 因为 1 + 2 + ... + n = n * (n + 1) / 2, 即（首 ＋ 尾）* 个数 ／ 2
            // 上面的价值就是 (cnt + remainMax + 1) * (cnt - (remainMax + 1) + 1) / 2
            res += (long)(cnt + remainMax + 1) * (long)(cnt - remainMax) / 2;
            res %= MOD;
            orders -= cnt - remainMax;
        }
        // b: 初始数量 ＝ 剩余数量的，随便挑几个卖掉 orders 个，当然这里的 orders 有可能已经为零。
        res += (long) remainMax * (long) orders;

        // 返回 a + b
        return (int) (res % MOD);
    }
    // 初始数量 > 剩余数量的同色球一个个卖掉，直到它的数量等于 remainMax，返回总共卖掉的球数
    private int getBallCnt(int[] inventory, int remainMax, int total) {
        int res = 0;
        for (int cnt : inventory) {
            if (cnt <= remainMax) continue;
            res += (cnt - remainMax);
            if (res > total) break;
        }
        return res;
    }
}
